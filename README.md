# GovAI Assistant (Norway) — PathBoot2

A privacy-first, local AI-powered government guidance system to help citizens—especially immigrants in Norway—navigate public services such as **NAV**, **tax**, and **immigration**. Users ask questions in natural language and get **clear, structured, multilingual** answers generated by a **local LLM** with **retrieval-augmented generation (RAG)**.

> Example: "How do I apply for unemployment benefits in Norway?"
> The assistant returns structured answers and supports at least **English + Norwegian + Amharic**.

## Project Overview

### Purpose & Impact
GovAI Assistant improves **digital inclusion and accessibility** by reducing the friction of navigating complex government websites and procedures—especially for newcomers facing language barriers.

**Key benefits**
- Makes Norwegian public services easier to understand (NAV, Tax, Immigration)
- Multilingual support (target: **English, Norwegian, Amharic**)
- Step-by-step guidance that is easier to follow than dense web pages
- Reduced dependency on support centers and long wait times
- Privacy-first approach by using **local inference** where possible

### What this repository contains
This repo is the **backend-first** implementation (Spring Boot + AI orchestration) designed to integrate with a modern chat UI (recommended stack below).

---

## Architecture (High-Level)

**Request flow**
1. User asks a question (UI / API)
2. Session memory stores conversation context
3. Language is detected / normalized
4. Domain router classifies question:
   - NAV (welfare)
   - Tax
   - Immigration
5. Agent factory creates domain-specialized agent
6. RAG retrieves relevant documents (vector embeddings + similarity search)
7. Prompt is composed (context + chat history + instructions)
8. Local LLM (e.g., **Mistral via Ollama**) generates answer
9. Response is translated back to user language
10. Response stored and returned

## MVP Definition

### MVP Goal
Deliver a working end-to-end assistant that:
- answers questions for at least **one domain** (NAV recommended first),
- uses **RAG** with a small curated dataset,
- runs **locally** (LLM + embeddings),
- **returns structured answers and supports at least English + Norwegian + Amharic**.

### MVP In-scope (recommended)
- Single chat endpoint with sessions
- Domain routing (NAV/Tax/Immigration)
- RAG retrieval + prompt composition
- Local inference via Ollama (Mistral)
- Basic translation layer (or bilingual prompts)

### Out of scope (for MVP)
- Production-grade auth + billing
- Large-scale document ingestion pipelines
- Full UI polish / mobile app
- Complex observability + multi-region infra

## Backend

### Core components
- **Spring Boot REST API**
- Orchestrator service (pipeline coordination)
- Domain router (classification)
- Agent factory + domain agents (prompt strategies)
- RAG:
  - Embeddings (e.g., `nomic-embed-text`)
  - Vector store (implementation-defined)

### API (suggested endpoints)
**Must have**
- `POST /api/chat` — send a message, receive an answer
- `GET /api/health` — liveness/readiness basics

**Should have**
- `POST /api/sessions` — create session
- `GET /api/sessions/{id}` — fetch history
- `POST /api/admin/ingest` — ingest/update documents (secured)

**Nice to have**
- `GET /api/metrics` — Prometheus metrics
- `POST /api/feedback` — user feedback loop (thumbs up/down + comment)

## Setup & Run (Local)

### Prerequisites
- Java 17+
- Maven
- Git
- Ollama installed and running

### 1) Start local LLM + embeddings (Ollama)
```bash
ollama pull mistral
ollama pull nomic-embed-text
ollama serve
```

### 2) Run backend
```bash
./mvnw spring-boot:run
```

Backend default:
- http://localhost:8080

### 3) Run frontend (if present)
```bash
cd frontend
npm install && npm run dev
```

Frontend default:
- http://localhost:3000

## Final Recommendation

1. **Ship a NAV-focused MVP first** with a small, high-quality dataset and strong answer structure (steps, requirements, links, next actions).
2. Implement **RAG + citations** early—trust is critical for government guidance.
3. Keep inference **local by default** (Ollama) and explicitly document privacy/retention assumptions.
4. Use Copilot to accelerate backlog creation and documentation, but treat it as an assistant—**not a source of truth**.
5. After MVP, prioritize streaming UX + ingestion workflows + automated tests to support safe iteration.
